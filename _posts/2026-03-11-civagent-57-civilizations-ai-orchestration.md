今天开源了一个我最近在做的项目——[CivAgent](https://github.com/LeoLin990405/CivAgent)。

一句话介绍：**把人类历史上 57 种经典政体，映射为 AI 多 Agent 协作的组织架构。** 像《文明》游戏一样，选择不同政体，体验不同的 AI 编排模式。零代码修改，纯配置切换。

## 起因

这学期在读钱穆先生的《中国历代政治得失》，一个核心观点反复出现：**每一代的制度都是对上一代制度弊端的回应。**

- 汉代三公九卿权力集中 → 唐代用三省来分权制衡
- 唐代门下省封驳权过大 → 宋代削弱相权、设二府三司
- 宋代文官制衡导致效率低下 → 明代创内阁提速，又用司礼监制衡内阁

这种「问题 → 制度回应 → 新问题 → 新制度」的迭代逻辑，跟软件架构演进一模一样。

然后我刷到了 [@wanikua](https://github.com/wanikua) 的 [AI 朝廷](https://github.com/wanikua/boluobobo-ai-court-tutorial)——用唐朝三省六部制来编排 AI Agent。这个想法直接点燃了我：**既然唐朝可以，为什么不把所有经典政体都做出来？**

## 6 种编排模式

从 57 种政体中提炼出 6 种核心编排模式：

| 模式 | 代表文明 | AI 编排特点 | 适用场景 |
|------|---------|------------|---------|
| **中央集权** | 秦、苏联、拿破仑 | 单 main agent 统管 | 紧急任务，快速决策 |
| **制衡** | 唐、宋、美国 | 多 agent 互审 | 高质量要求，防错 |
| **双轨** | 明（内阁+司礼监）、斯巴达 | 双 main 交叉审批 | 双重审批流程 |
| **联邦** | 周、三国、波斯 | 独立 agent 组自治 | 多团队并行 |
| **民主议会** | 雅典、维京、瑞士 | 全体 agent 投票 | 头脑风暴，集体决策 |
| **神权** | 商、古埃及、拜占庭 | 绝对权威 | 强力领导，危机处理 |

## 制度设计 ≈ 分布式系统架构

这不是牵强附会。古代政治制度和现代 Agent 编排面对的核心问题是一样的——信息不对称、委托-代理困境、效率与质量的平衡：

| 古代制度 | Agent 编排 | 共同解决的问题 |
|---------|-----------|-------------|
| 秦·三公九卿 | 单 main agent | 快速决策、一致性 |
| 唐·三省六部 | 多 agent 交叉审核 | 防止单点错误 |
| 明·内阁+司礼监 | 双 main 交叉审批 | 权力制衡 |
| 周·宗法分封 | 独立 agent 组 | 大规模并行 |
| 雅典·公民大会 | 等权 agent 投票 | 集体智慧 |

福山在《政治秩序的起源》中提出的三个维度——国家能力、法治、民主问责——恰好对应 AI 编排中的三个核心权衡：**执行效率、错误预防、参与度**。这三者不可能同时最大化。57 种政体，就是人类对这个「不可能三角」的 57 种不同回答。

## 怎么用

```bash
# 克隆
git clone https://github.com/LeoLin990405/civagent.git
cd civagent

# 列出 57 种文明
./scripts/list-regimes.sh

# 切换政体（一行命令）
./scripts/switch-regime.sh china/qin       # 秦·中央集权
./scripts/switch-regime.sh global/athens   # 雅典·直接民主
./scripts/switch-regime.sh china/ming      # 明·双轨制
```

每种政体是 5 个配置文件（JSON + Markdown），切换不需要改任何代码。
